# K-Volley Lab Project Rules

K-Volley Lab은 한국 배구 데이터를 더 쉽게 찾고, 비교하고, 분석할 수 있게 만드는 웹 기반 배구 데이터 플랫폼이다.

이 저장소에서 작업할 때는 아래 원칙을 우선한다.

## 1. 프로젝트 목표

K-Volley Lab의 1차 목표는 사람들이 자주 방문할 수 있는 실용적인 배구 정보 허브를 만드는 것이다.

초기 핵심 기능은 다음과 같다.

- 국내 배구 선수 성장 DB
- VNL 및 국제대회 선수 DB
- 대회 일정 달력
- 대회 팜플렛 아카이브
- 본선 진출 경우의 수 계산기
- 선수 검색 및 프로필 페이지
- 제보 및 수정 요청 접수

장기적으로는 한국형 Volleybox + 전력분석 자료 허브를 지향한다.

## 2. 데이터 운영 원칙

K-Volley Lab의 데이터는 공개적으로 확인 가능한 자료를 기반으로 한다.

사용 가능한 주요 출처 예시는 다음과 같다.

- 대회 공식 팜플렛
- 대한배구협회, KOVO, AVC, Volleyball World 등 공식 자료
- 학교, 팀, 협회, 연맹의 공개 게시물
- 공개 기사
- 공식 경기 기록지
- 선수 또는 팀의 공개 프로필
- Volleybox 등 배구 관련 공개 데이터베이스

중요한 데이터는 가능한 한 출처를 함께 저장한다.

## 3. 개인정보 보호 원칙

아래 정보는 수집하거나 공개하지 않는다.

- 주민등록번호
- 개인 연락처
- 개인 주소
- 비공개 가족정보
- 의료정보 중 공식적으로 공개되지 않은 세부 내용
- 선수 본인 또는 공식 출처가 공개하지 않은 민감 정보

선수 성장 DB는 공개 팜플렛, 공식 자료, 기사 등으로 확인 가능한 범위 안에서만 구성한다.

## 4. 선수 정보 표시 원칙

선수 정보는 가능하면 다음 항목을 포함한다.

- 영문명
- 한글명
- 국적
- 포지션
- 키
- 생년월일 또는 생년
- 소속팀
- 대표팀 여부
- 출처 링크
- Volleybox 프로필 링크

해외 선수 표시 시에는 가능하면 다음 형식을 따른다.

```text
Poriya Khanzadeh (OH) 🇮🇷
(포리야 칸자데, 이란, 196cm)
```

키가 확인되지 않으면 `키 확인 불가`로 표시한다.

## 5. 파일명 및 버전 관리 원칙

사용자에게 수정 파일을 제공하거나 저장소에서 버전성 작업을 할 때는 긴 설명을 파일명 뒤에 붙이지 않는다.

권장 방식:

```text
파일명_v1
파일명_v2
파일명_v3
```

K-Volley Lab 선수 공용 코드는 `KVL`을 사용한다.
신규 선수 ID가 필요할 때는 가능한 한 다음 형식을 따른다.

```text
INITIALS-YYMMDD
```

## 6. 디자인과 기능 작업 원칙

새 페이지나 기능을 만들 때는 기존 K-Volley Lab의 방향과 디자인을 유지한다.

작업 전 확인 우선순위는 다음과 같다.

1. `PROJECT_RULES.md`
2. `DESIGN_SYSTEM.md`
3. `STRUCTURE.md`
4. `TODO.md`
5. 기존 HTML, CSS, JS 파일

새 기능을 추가할 때 기존 메뉴, 색상, 카드 스타일, 모바일 반응형 구조가 깨지지 않도록 한다.

## 7. 홈페이지 성격

K-Volley Lab은 단순 팬 페이지가 아니라, 전력분석관 관점의 데이터 정리와 배구 인프라 확장을 목표로 한다.

따라서 콘텐츠는 다음 기준을 우선한다.

- 보기 쉬운 정보
- 출처가 남는 데이터
- 선수와 대회를 연결하는 구조
- 모바일에서도 빠르게 확인 가능한 화면
- 현장 관계자, 선수, 팬 모두가 이해할 수 있는 표현

## 8. 금지 또는 주의 작업

아래 작업은 신중하게 처리한다.

- 출처 없는 확정 표현
- 확인되지 않은 이적, 부상, 신상 정보 단정
- 페이지마다 다른 디자인 체계 추가
- 같은 기능의 중복 페이지 무분별 생성
- JSON 데이터 구조와 화면 코드 불일치
- 기존 링크 구조를 검토하지 않은 파일명 변경

## 9. 작업 방식

대규모 구조 변경 전에는 먼저 현재 구조를 점검한다.

가능하면 다음 순서로 작업한다.

1. 현재 파일 구조 확인
2. 영향받는 페이지 확인
3. 수정 계획 정리
4. 파일 수정
5. 연결 링크 확인
6. 다음 작업 TODO 반영

K-Volley Lab은 장기 프로젝트이므로, 빠른 임시 수정만큼이나 유지보수 가능한 구조를 중요하게 본다.
